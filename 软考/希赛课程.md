#### 信息系统生命周期
1. 产生阶段：提出想法，对需求进行调研和分析
2. 开发阶段：总体规划 => 系统分析 => 系统设计 => 系统实施 => 系统验收
3. 运行阶段：验收、移交之后，系统维护
4. 消亡阶段：更新改造、重构、功能扩展等

#### 信息系统建设原则
1. 高层管理人员介入原则（cio（既懂计算又懂管理）介入，解决阻碍问题）
2. 用户参与开发原则（用户确定范围）、核心用户全程参与、用户深度参与
3. 自顶向下规划原则（以此减少信息不一致的现象）
4. 工程化原则（引入软件工程）
5. 其它原则

#### 信息系统开发方法

1. 结构化法（自顶向下）
- -- 自顶向下，逐步分解求精
- -- 开发目标清晰化
- -- 工作阶段程式化
- -- 开发文档规范化
- -- 设计方法结构化
- -- 应变能力差

2. 面向对象方法（自底向上）
- -- 自底向上
- -- 阶段界限不明
- -- 更好应变、更好复用
- -- 符合人们的思维习惯

3. 面向服务的方法（包含面向对象）
- -- 粗粒度、松耦合
- -- 标准化和结构化
- -- 抽象级别：操作（低） -> 服务（中） -> 业务流程（高）

3. 原型开发方法（既是方法也是开发模型）
- -- 构建简易系统，来获取需求
- 分抛弃型原型（得到需求后就丢掉，按需求重新开发）和演化型原型（由原型逐步演化成最终系统）

4. 形式化方法（数学化模型）
- -- 净室软件工程
- -- 所有东西均可证明/验证，而不是测试

5. 统一过程方法（up）

- -- 既是系统开发方法，也是一种开发模型

#### 信息系统分类（可能考1~2分）
1. 业务处理系统（TPS）（数据流风格）（已经不用了1950-1960年代的产物）
- -- 早期最初级的信息系统
- -- 功能：数据输入、数据处理（批处理、OLTP）、数据库维护、文件报表产生
- -- TPS是服务于组织管理层次中最底层、最基础的信息系统

2. 管理信息系统（MIS）（已经不火了）
- -- 高度集成化的人机信息系统
- -- 金字塔结构：分多个层次
- -- 批处理系统（属于开环系统，就是在决定过程中，不断收集信息，不断发送给决策者）
- -- 计算机实时处理系统（属于闭环系统）

3. 决策支持系统（DSS）（已经不火了）
- -- 由语言系统、知识系统和问题处理系统组成。用于辅助决策，支持决策
- -- 计算机实时处理系统（属于闭环系统）

4. 专家系统（ES）
- -- ES与Dss的重要区别（ES会进行判断与决策）
- -- 包括：知识库、综合数据库、推理机、知识获取、解释程序（面向用户服务的）
- -- 知识 + 推理 = 专家系统，人工智能是一个重要分支

5. 办公自动化系统（OAS）
- -- 由计算机设备、办公设备、数据通信及网络设备、软件系统组成

6. 企业资源计划（ERP）
- -- 打通供应链，集成，整合
