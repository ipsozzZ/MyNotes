
# 信息系统知识

## # 信息系统概述
信息系统是由计算机硬件、网络和通信设备、计算机软件、信息资源、信息用户和规章制度组成的以处理信息流为目的的人机一体化系统。

信息系统的5个基本功能：输入、存储、处理、输出和控制

#### 诺兰模型
1. 初始阶段
2. 传播阶段
3. 控制阶段
4. 集成阶段
5. 数据管理阶段
6. 成熟阶段

1~6阶段逐步递进

#### 信息系统的分类
1. 业务处理系统（TPS/DPS）（数据流风格）（已经不用了1950-1960年代的产物）
- -- 早期最初级的信息系统
- -- 功能：数据输入、数据处理（批处理、OLTP）、数据库维护、文件报表产生
- -- TPS是服务于组织管理层次中最底层、最基础的信息系统
- -- 如自动售票机等

2. 管理信息系统（MIS）（已经不火了）
- -- 高度集成化的人机信息系统
- -- 金字塔结构：分多个层次
- -- 批处理系统（属于开环系统，就是在决定过程中，不断收集信息，不断发送给决策者）
- -- 计算机实时处理系统（属于闭环系统）

3. 决策支持系统（DSS）（已经不火了）
- -- 由语言系统、知识系统和问题处理系统组成。用于辅助决策，支持决策
- -- 计算机实时处理系统（属于闭环系统）

4. 专家系统（ES）
- -- ES与Dss的重要区别（ES会进行判断与决策）
- -- 包括：知识库、综合数据库、推理机、知识获取、解释程序（面向用户服务的）
- -- 知识 + 推理 = 专家系统，人工智能是一个重要分支

5. 办公自动化系统（OAS）
- -- 由计算机设备、办公设备、数据通信及网络设备、软件系统组成

6. 企业资源计划（ERP）
- -- 打通供应链，集成，整合

7. 其它

#### 信息系统生命周期
1. 生产：概念产生、需求分析，并形成需求分析报告

2. 开发：包括总体规划、系统分析、系统设计（系统架构设计、数据库设计、功能模块设计、安全机制设计等）、系统实施（编码、验收，用户参与）和系统验收5个阶段

3. 运行：运行、维护（运维）

4. 消亡

#### 信息系统的开发方法（软件工程部分也有这个内容，这里简单了解）

1. 结构化方法的主要特点：
结构化方法。结构是指系统内各个组成要素之间的相互联系、相互作用的框架。结构化方法是一种传统的信息系统开发方法，由结构化分析（SA）、结构化设计（SD）和结构化程序设计（SP）三部分有机组合而成，其精髓是自顶向下、逐步求精和模块化设计。
- -- 开发目标清晰化
- -- 开发工作阶段化
- -- 开发文档规范化
- -- 设计方法结构化

2. 结构化方法的不足和局限：
- -- 开发周期长
- -- 难以适应需求变化
- -- 很少考虑数据结构

1. 结构化法（自顶向下）
- -- 自顶向下，逐步分解求精
- -- 开发目标清晰化
- -- 工作阶段程式化
- -- 开发文档规范化
- -- 设计方法结构化
- -- 应变能力差

2. 面向对象方法（自底向上）
- -- 自底向上
- -- 阶段界限不明
- -- 更好应变、更好复用
- -- 符合人们的思维习惯

3. 面向服务的方法（包含面向对象）
- -- 粗粒度、松耦合
- -- 标准化和结构化
- -- 抽象级别：操作（低） -> 服务（中） -> 业务流程（高）

3. 原型开发方法（既是方法也是开发模型）
- -- 构建简易系统，来获取需求
- 分抛弃型原型（得到需求后就丢掉，按需求重新开发）和演化型原型（由原型逐步演化成最终系统）

4. 形式化方法（数学化模型）
- -- 净室软件工程
- -- 所有东西均可证明/验证，而不是测试
